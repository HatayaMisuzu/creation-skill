# Interaction Performance Guide

Use this for immersive character dialogue, OOC repair, prompt-card writing, and
project simulation frontstage prose.

## Default Reply Shape

- Respond to the user's latest message first.
- Use character dialogue as the center.
- Add short actions only when they reveal emotion or scene movement.
- Avoid long narration unless the user asked for a scene.
- End with a natural hook, not a questionnaire checklist.

## Language Consistency

- Match the user's current language.
- For Chinese users, use Chinese as the main reply language.
- Keep Japanese/English only for short catchphrases, names, titles, honorifics, or source-specific terms.
- Do not switch into source language just because evidence is in that language.

## Immersion Rules

Never show these in user-facing roleplay or scene prose:

- numeric relationship or emotion deltas
- backend state
- `debug`
- `tension`
- `scene_focus`
- speaker schedules
- evaluator labels
- hidden chain-of-thought or source tables

## OOC Repair Method

When repairing a bad reply:

1. Identify the failure: voice drift, language mismatch, profile recitation, meta leak, relationship over-expansion, weak refusal, wrong emotion, or scene mismatch.
2. Name the violated card rule briefly.
3. Rewrite the reply in character.
4. Add one short future rule if the card should be updated.

## Voice Differentiation

For multi-character scenes, separate:

- sentence length
- interruption tendency
- silence tendency
- emotional openness
- address style
- conflict reaction
- humor style

The user should feel different people speaking, not one narrator wearing names.
